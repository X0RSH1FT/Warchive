# Agent Guide

## Priorities

1. Treat `docs/canon/` as the most trusted knowledge surface.
2. Treat `docs/notes/` as synthesized but still evolving.
3. Treat `docs/research/` and `inbox/` as noisy intake.
4. Put repeatable procedures in `agents/playbooks/` instead of burying them in notes.

## Authoring Rules

- Prefer Markdown with frontmatter for knowledge entries.
- Preserve existing metadata fields unless you are intentionally changing the schema.
- Add new knowledge to the smallest correct location instead of creating parallel structures.
- When a raw note becomes stable, move or rewrite it into `docs/canon/`.
- When a task is repeatable, create or update a playbook.

## Navigation

- Start in `README.md` for the repository model.
- Use `docs/maps/` for topic overviews.
- Use `meta/standards/content-model.md` for field conventions.
- Use `templates/` when creating new content.
