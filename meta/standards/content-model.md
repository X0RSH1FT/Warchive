# Content Model

Use these fields for Markdown knowledge entries when practical.

## Core Fields

- `title`: human-readable title
- `type`: one of `note`, `canon`, `research`, `playbook`, or `map`
- `status`: lifecycle marker such as `seed`, `working`, `curated`, or `archived`
- `tags`: list of classification tags
- `updated`: ISO date of the last meaningful edit
- `source`: `internal`, `external`, or a more specific origin
- `confidence`: `low`, `medium`, or `high`

## Rules

- Use `seed` for quick captures and early drafts.
- Use `working` for active material that still needs review.
- Use `curated` for entries that are stable enough to trust.
- Use `archived` when an entry is retained for history but should not guide new work.

## Guidance

- Keep one primary idea per file.
- Prefer explicit titles over clever ones.
- Use tags to group by subject, system, or source family.
- Promote stable knowledge from research to notes, then to canon when justified.
