# Tag Taxonomy

Keep tags short, predictable, and composable.

## Suggested Namespaces

- `domain/`: subject matter such as `domain/ai` or `domain/security`
- `system/`: repository-level markers such as `system/index`
- `source/`: upstream origin such as `source/github` or `source/book`
- `workflow/`: recurring task type such as `workflow/research`
- `tool/`: tool or platform such as `tool/copilot`

## Rules

- Prefer lowercase tags.
- Prefer slash-delimited namespaces for consistency.
- Add new namespaces sparingly.
- Reuse existing tags before inventing close variants.
